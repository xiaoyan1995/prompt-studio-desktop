## 一、产品定位

### 1.1 一句话描述

在 Prompt Studio 桌面端左侧导航栏新增「🎬 智能拉片」入口，作为与「所有项目」并列的**独立子系统**。用户拖入本地视频后，系统自动完成「物理分镜 → 关键帧提取 → AI 视听解构 → AIGC 提示词反推」全流程，输出可直接用于 Sora / Kling / FLUX 等平台的影视级提示词。

### 1.2 目标用户

- AIGC 短视频创作者（需要拆解高质量参考片，反推提示词）
- 影视/广告从业者（需要对样片进行专业拉片分析）
- 提示词工程师（需要把视觉灵感转化为精准的文字控制指令）

### 1.3 核心优势（对比 FM80cine 网页版）

| 维度 | FM80cine (Web) | Prompt Studio (本地桌面端) |
|------|---------------|--------------------------|
| 视频大小限制 | 最大 300MB | **无限制**（本地读取，不上传） |
| 分镜速度 | 需要上传等待 + 服务器排队 | **毫秒级**（本地 FFmpeg） |
| AI 分析费用 | 平台承担（限制次数） | **用户自己的 API Key**（无限次） |
| 数据隐私 | 视频上传至第三方服务器 | **纯本地处理**，仅关键帧小图发给 AI |
| 与资产库联动 | 无 | 拉片结果可一键存入项目资产库 |

### 3.2 模块二：本地分镜引擎（FFmpeg 物理层）

#### 3.2.1 场景切点检测

**触发时机**：用户点击「开始拉片」后，系统自动启动。

**技术实现**：
```
后端 server.py 新增接口: POST /api/lapian/detect-scenes
```

**处理流程**：
1. 检查本地 FFmpeg 是否可用（调用现有 `find_ffmpeg()`）。
2. 若 FFmpeg 未安装，弹出「需要安装 FFmpeg」的提示（复用现有下载安装逻辑）。
3. 执行场景检测命令：
   ```
   ffprobe -show_frames -of csv -f lavfi \
     "movie={video_path},select=gt(scene\,{threshold})" \
     2>/dev/null | grep "video" | cut -d',' -f5
   ```
   - `threshold` 默认值 `0.3`（可在设置中调整）。
   - 输出：所有场景切换点的精确时间戳数组。
4. 根据切点时间戳，计算出每个镜头的 `start_time`、`end_time`、`duration`。
5. 将结果写入本地 JSON 文件。

**接口定义**：
```
POST /api/lapian/detect-scenes
请求体: { "videoPath": "D:/videos/zombie.mp4", "threshold": 0.3 }
响应体: {
  "success": true,
  "totalDuration": 120.5,
  "shots": [
    { "shotNumber": 1, "startTime": 0.0, "endTime": 3.2, "duration": 3.2 },
    { "shotNumber": 2, "startTime": 3.2, "endTime": 7.5, "duration": 4.3 },
    ...
  ]
}
```

**前端状态展示**：
- 进度条 + 文字提示：「正在分析视频场景... (12/120s)」
- 预计耗时：10 秒视频 ≈ 200ms，2 小时电影 ≈ 10-30 秒。

#### 3.2.2 关键帧提取

**触发时机**：场景检测完成后，自动执行。

**技术实现**：
```
后端 server.py 新增接口: POST /api/lapian/extract-keyframes
```

**处理流程**：
对于每一个检测出的镜头片段，使用 FFmpeg 截取 **3 张关键帧**：
- **First Frame（首帧）**：`start_time + 0.05s`（避免转场黑帧）
- **Middle Frame（中间帧）**：`start_time + duration / 2`
- **Last Frame（末帧）**：`end_time - 0.05s`

```
ffmpeg -ss {timestamp} -i {video_path} -vframes 1 -q:v 2 \
  -vf "scale=640:-1" {output_path}
```

**输出目录结构**：
```
{DATA_DIR}/lapian_data/{project_id}/
├── meta.json              ← 项目元数据
├── shots/
│   ├── shot_001_first.jpg
│   ├── shot_001_mid.jpg
│   ├── shot_001_last.jpg
│   ├── shot_002_first.jpg
│   ├── shot_002_mid.jpg
│   ├── shot_002_last.jpg
│   └── ...
└── analysis.json          ← AI 分析结果
```

**接口定义**：
```
POST /api/lapian/extract-keyframes
请求体: { "projectId": "uuid", "videoPath": "...", "shots": [...] }
响应体: {
  "success": true,
  "keyframes": [
    {
      "shotNumber": 1,
      "first": "/lapian_data/{id}/shots/shot_001_first.jpg",
      "middle": "/lapian_data/{id}/shots/shot_001_mid.jpg",
      "last": "/lapian_data/{id}/shots/shot_001_last.jpg"
    },
    ...
  ]
}
```

**前端状态展示**：
- 进度条 + 文字提示：「正在提取关键帧... (5/15 个镜头)」
- 每提取完一个镜头的关键帧，立即在前端渲染出该镜头的缩略图卡片（渐进式加载体验）。

### 3.4 模块四：AI 多模态分析引擎

#### 3.4.1 批量自动分析

**触发时机**：关键帧提取完成后，顶部出现横幅按钮「🤖 AI 自动分析所有镜头」。

**处理流程**：
1. 前端逐个（或批量）将每个镜头的 Middle Frame 图片转为 Base64。
2. 调用后端接口 `POST /api/lapian/ai-analyze`。
3. 后端读取用户在「设置」中配置的大模型 API Key（OpenAI / Gemini / 其他兼容接口）。
4. 发送多模态请求，System Prompt 采用影视学院级拉片 Agent 提示词（见 §3.4.3）。
5. 解析 AI 返回的结构化 JSON，写入本地 `analysis.json`。
6. 前端实时更新：每完成一个镜头的分析，立即填充到对应卡片中（流式体验）。

**接口定义**：
```
POST /api/lapian/ai-analyze
请求体: {
  "projectId": "uuid",
  "shotNumber": 1,
  "images": ["base64_of_mid_frame"],
  "model": "gpt-4o"
}
响应体: {
  "success": true,
  "analysis": {
    "framing": "中景 (Medium Shot)",
    "cameraAngle": "平视 (Eye-level)",
    "cameraMovement": "跟拍 (Tracking Shot)",
    "visualDescription": "一名身穿黑色战术装备的男子...",
    "atmosphereColor": "暗调冷色，蓝绿霓虹灯光...",
    "narrativeFunction": "交代主角的行动场景，建立紧张氛围...",
    "aigcPrompt": "A medium tracking shot of a tactical-geared man..."
  }
}
```

**前端状态展示**：
- 分镜缩略图条上的每个卡片显示分析进度：
  - 灰色圆点 ⚪：未分析
  - 蓝色旋转 🔄：分析中
  - 绿色对勾 ✅：已完成
- 顶部横幅：「AI 正在分析... (5/15 个镜头)」

#### 3.4.2 单镜头重新分析

用户在镜头详情面板中，可以点击「🤖 AI 重新分析此镜头」，仅对当前选中的镜头重新发起 AI 请求。支持发送 1 张（Middle）或 3 张（First + Middle + Last）图片进行更精确的分析。

#### 3.4.3 AI Agent System Prompt（核心提示词）

```

请分析下面的电影/视频关键帧图片，并严格以如下 JSON 格式返回分析结果（不要包含任何 Markdown 标记或额外解释文字）：

{
  "framing": "从以下选项中选择最匹配的：极特写(ECU)/特写(CU)/中近景(MCU)/中景(MS)/中远景(MLS)/远景(LS)/全景(WS)/大全景(EWS)",
  "cameraAngle": "从以下选项中选择：平视(Eye-level)/仰拍(Low-angle)/俯拍(High-angle)/鸟瞰(Bird's-eye)/荷兰角(Dutch Angle)",

  "aigcPrompt": "用英文生成一段可直接用于 Sora/Midjourney/FLUX 的影视级提示词，包含景别、运镜、光影、主体、氛围（50-120 words）"
}
```

### 3.5 模块五：分镜手动调整

#### 3.5.1 拆分镜头

**场景**：AI 检测的切点不精确（例如把一个渐变转场内的两个镜头合并成了一个）。

**操作**：
1. 在主播放器中逐帧定位到需要拆分的精确位置。
2. 点击工具栏「✂️ 在此拆分」按钮（或快捷键 `S`）。
3. 当前镜头被一分为二，自动为新镜头提取关键帧。
4. 分镜缩略图条实时更新。

**接口**：
```
POST /api/lapian/split-shot
请求体: { "projectId": "uuid", "shotNumber": 3, "splitTime": 5.8 }
```

#### 3.5.2 合并镜头

**场景**：AI 把一个连续运镜（如一个长推镜头）错误地拆成了多个碎片。

**操作**：
1. 在缩略图条中多选相邻的 2-N 个镜头。
2. 点击工具栏「🔗 合并」按钮。
3. 合并后自动重新提取关键帧。

**接口**：
```
POST /api/lapian/merge-shots
请求体: { "projectId": "uuid", "shotNumbers": [3, 4, 5] }
```

#### 3.5.3 替换关键帧

**场景**：自动抽的关键帧恰好是模糊帧或黑帧。

**操作**：
1. 在镜头详情面板的三帧预览区，右键某张图片 → 「替换关键帧」。
2. 弹出视频迷你播放器，用户逐帧定位到理想画面。
3. 点击「截取此帧」，替换原有关键帧。

**接口**：
```
POST /api/lapian/replace-keyframe
请求体: { "projectId": "uuid", "shotNumber": 1, "position": "middle", "timestamp": 1.85 }
```

#### 3.5.4 手动添加镜头

**场景**：AI 漏检了一个镜头（例如：叠化转场中的两个画面 AI 没识别为独立镜头）。

**操作**：
1. 在主播放器中定位到漏检的镜头起始位置。
2. 点击工具栏「+ 添加镜头」，标记起始时间。
3. 继续播放到该镜头结束位置，点击「确认结束」。
4. 系统自动提取关键帧并插入到分镜条中。

#### 3.5.5 重新分镜

**场景**：用户调整了灵敏度后，希望全部重来。

**操作**：
1. 点击工具栏「🔄 重新分镜」。
2. 弹窗：选择灵敏度（低/中/高，对应 threshold 0.4/0.3/0.2）。
3. 确认后重新执行场景检测 + 关键帧提取。
4. **注意**：重新分镜会清除所有手动编辑和 AI 分析结果，需二次确认。

### 3.6 模块六：导出与联动

#### 3.6.1 导出分镜报告

**格式选项**：
1. **Markdown 报告**（`.md`）：
   - 项目标题 + 视频信息
   - 每个镜头：关键帧图片（内嵌） + 景别/运镜/描述/提示词
   - 适合在 Obsidian / Notion 中查阅

2. **JSON 数据**（`.json`）：
   - 完整的结构化数据
   - 适合二次开发或导入其他工具

3. **CSV 表格**（`.csv`）：
   - 镜号 | 时间 | 景别 | 角度 | 运镜 | 描述 | 提示词
   - 适合在 Excel 中查看和标注

4. **图片长图**（`.png`）：
   - 竖版拼接所有镜头的 Key Frame + 标签
   - 适合社交媒体分享或存档

**接口**：
```
POST /api/lapian/export
请求体: { "projectId": "uuid", "format": "markdown" }
响应体: { "filePath": "exports/lapian_丧尸清道夫_2026-05-21.md" }
```

#### 3.6.2 导出到资产库

将拉片中的某个镜头或整个项目的分析结果，一键存入现有的「所有项目」资产库中：
- 选中一个镜头 → 点击「存入资产库」→ 选择目标项目 → 创建一条新的提示词资产，内容为该镜头的 AIGC 提示词 + 关键帧图片作为封面。
- 或选择「批量导出所有镜头」，为每个镜头各创建一条资产。

#### 3.6.3 导出提示词合集

一键导出所有镜头的 AIGC 提示词为纯文本文件，每行一个提示词，方便批量使用。

### 3.7 模块七：在线视频下载

#### 3.7.1 通过链接下载

**场景**：用户想对网上看到的一个视频进行拉片，但视频还没下载到本地。

## 六、设置项

在现有「设置」面板中新增「拉片设置」区域：

| 设置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 分镜灵敏度 | 滑块 (0.1 - 0.5) | 0.3 | 值越小越敏感，切出的镜头越多 |
| 关键帧质量 | 下拉 | 高 (q=2) | 高/中/低，影响关键帧图片清晰度 |
| 关键帧宽度 | 数字 | 640px | 抽取的关键帧图片宽度 |
| AI 分析模型 | 下拉 | gpt-4o | 从用户已配置的模型列表中选择 |
| 提示词语言 | 下拉 | 英文 | 英文/中文/中英混合 |
| 分析精度 | 下拉 | 标准(1帧) | 标准(发送 Middle 1帧)/精细(发送 3帧) |

## 九、技术风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|---------|
| FFmpeg 未安装 | 无法分镜 | 复用现有自动下载安装逻辑，首次使用时引导安装 |
| 超长视频（>2h）分镜慢 | 用户等待 | 后台异步处理 + 进度反馈；分批提取关键帧 |
| AI API 超时或报错 | 分析中断 | 支持单镜头重试；批量分析失败不阻塞其他镜头 |
| 大量关键帧占磁盘 | 存储压力 | 默认 640px 低分辨率；提供「清理缓存」功能 |
| HTML5 Media Fragments 兼容性 | Hover 预览失效 | Electron Chromium 内核完美支持，无兼容问题 |
